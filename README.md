#基于epoll的reactor服务器

##采用C++标准
C++17

##采用的库
1. 网络套节字，采用linuxC++原生api，即巴克利socket。
2. json库为rapidjson，已集成在./inlcude中，用来读取配置文件或者用作数据传输（可能）。（现utils库）
3. 日志库使用的是C++原生ifstream，自封。（现utils库）

   
##设计思路
1. 设计库时尽量少的对外暴露接口。
2. 本网络服务器默认为长连接。
3. 对于指针对象的生命周期，在设计时应考虑，指针的创建在外部，库的内部负责指针的销毁、回收。
4. 使用多线程时，应考虑性能留有余量。
